# Bug Report: `npm run build:plugin` fails — missing manifest catalog prerequisite

**Date:** 2026-05-13  
**Command:** `npm run build:plugin`  
**Exit code:** 1  
**Failing step:** `validate-plugin-tree`

---

## Symptom

```
[build:plugin] validate-plugin-tree FAIL — missing:
  manifests/v1.0.0-alpha.8.json

Command exited with code 1
```

The build completes all earlier steps successfully (cli-build, pipeline-bundle, ui-standalone, adapters-plugin, cli-bundle, copy-shared-config, copy-manifest-catalog, copy-bundles-into-claude-plugin, copy-plugin-package-json, sync-plugin-version) but fails at the final validation assertion.

---

## Root cause

`build-plugin.js`'s `copy-manifest-catalog` step (step 7) reads from:

```
installer/src/claude/manifests/v<version>.json
```

It copies that file into the staged plugin tree at:

```
cli/dist/marketplaces/claude/plugins/rad-orchestration/manifests/v<version>.json
```

`validatePluginTree` then asserts that file exists and fails when it doesn't.

**The problem:** `installer/src/` is never committed to git and is not generated by `build:plugin`. It is a publish-time artifact produced exclusively by `installer/scripts/sync-source.js` (invoked via `npm pack`'s `prepack` hook in the installer package). On a fresh clone — or in a worktree that has never run the installer's publish flow — `installer/src/claude/manifests/` simply does not exist.

The `copy-manifest-catalog` step handles this silently: it checks `if (!fs.existsSync(srcCatalog)) return;` and exits without error, leaving the `manifests/` directory in the staged plugin tree empty. `validatePluginTree` then catches the omission.

**Confirmed state in this worktree:**

- `installer/src/` — does not exist (zero commits in `git ls-files installer/src`)
- `installer/.gitignore` — explicitly ignores `src/*/agents/`, `src/*/skills/`, `src/*/package.json`, `src/ui/`, but comments that `src/<harness>/manifests/` is **not** ignored and is supposed to be committed (append-only catalog)
- `installer/src/claude/manifests/` — never populated; no `v1.0.0-alpha.8.json` present

---

## What's supposed to happen

Per `installer/.gitignore` (lines near the bottom):

> `src/<harness>/manifests/` is NOT ignored. The manifest catalog is committed and grows append-only — every release adds a new `v<version>.json`, and prior versions stay so the installer can do clean upgrades.

This implies `installer/src/claude/manifests/v1.0.0-alpha.8.json` **should be committed** in the repo. It is absent in this worktree — either it was never generated and committed for this version, or the worktree branch predates the file being added.

---

## Secondary issue encountered

Before reaching the manifest failure, the build also failed on the first attempt because `cli/node_modules/` was absent, causing `tsc` (called by `npm run build` in the `cli-build` step) to be unresolvable:

```
'tsc' is not recognized as an internal or external command
[build:plugin] FAILED: Command failed: npm run build
```

`cli/node_modules/` is gitignored. Running `npm install` inside `cli/` unblocked this step. The `build-plugin.js` script does not check for or auto-install `cli/node_modules/` before invoking `npm run build`, unlike `installer/scripts/sync-source.js` which has an explicit `ensureCliBuilt()` guard.

---

## Reproduction steps

1. Fresh clone / worktree where `installer/src/` and `cli/node_modules/` do not exist
2. `npm run build:plugin` from repo root

**Step A failure** (missing `cli/node_modules/`):
```
'tsc' is not recognized as an internal or external command
[build:plugin] FAILED: Command failed: npm run build
```

Workaround: `cd cli && npm install`

**Step B failure** (missing manifest, after workaround A):
```
[build:plugin] validate-plugin-tree FAIL — missing:
  manifests/v1.0.0-alpha.8.json
```

No workaround without running `installer/scripts/sync-source.js` or manually creating the file.

## Affected files

| File | Role |
|---|---|
| `scripts/build-plugin.js` | Orchestrates the build; `copy-manifest-catalog` step silently no-ops if `installer/src/claude/manifests/` absent |
| `installer/scripts/sync-source.js` | Generates `installer/src/<harness>/manifests/v<version>.json`; publish-time only |
| `installer/.gitignore` | Documents that `src/<harness>/manifests/` is committed — but it isn't present in this worktree |
| `installer/src/claude/manifests/v1.0.0-alpha.8.json` | **Missing** — the specific file required by `validatePluginTree` |

## Suggested fixes (not implemented)

**Option A — Commit the missing manifest file**  
Generate `installer/src/claude/manifests/v1.0.0-alpha.8.json` via `installer/scripts/sync-source.js` (or a lighter `emitBundles` call) and commit it. This is what `installer/.gitignore` implies should exist.

**Option B — Add `ensureCliBuilt` guard to `build-plugin.js`**  
Mirror the `ensureCliBuilt()` pattern from `sync-source.js` so the `cli-build` step auto-installs `cli/node_modules/` on a cold clone instead of failing on `tsc`.

**Option C — Wire `sync-source.js` as a prerequisite of `build:plugin`**  
Run the manifest-generation portion of `sync-source.js` as a pre-step in `build:plugin` so `installer/src/claude/manifests/` is always populated before `copy-manifest-catalog` runs.
