Claude Code — Telemetry Surface

Where Claude Code emits data, and how to trace token spend to a subagent · verified 2026-05-24 against Claude Code 2.1.150

Five channels carry data off a session. Three correlation keys stitch them together. Three paths trace tokens to a specific subagent.

The joins

Three correlation keys

requestId

JSONL ↔ OTel

Top-level requestId in the JSONL transcript equals request_id on OTel api_request / api_error events.

transcript↔OTel
tool_use_id

OTel ↔ hooks

On OTel tool_result / tool_decision it matches the field passed to hooks on stdin.

OTel↔hooks
prompt.id

OTel events only

UUID v4 on events (intentionally not on metrics). Links every event triggered while processing one user prompt.

one prompt→all its events

The payoff

Three paths to per-subagent token attribution

1

OTel metrics agent.namemetrics

Built-in and official-plugin agents appear verbatim on token.usage / cost.usage; custom user-defined agents collapse to custom unless OTEL_LOG_TOOL_DETAILS=1. Zero pipeline code.

2

OTel traces agent_id / parent_agent_idtraces

Span-level, unconditional once traces are on. Subagent llm_request spans nest under the parent's tool span and carry token counts. Cleanest answer if a trace backend exists.

3

JSONL correlationlocal disk

Flat-rate, no collector: sum message.usage across subagents/agent-<hex>.jsonl; the .meta.json gives the agent type.

Note: the old "per-subagent tracking is a feature gap" claim (issue #22625) predates this agent.name / agent_id / subagents/ layout.

Flat-rate plans (Pro / Max)

  • No Console analytics — that's Team / Enterprise only. Parse your own JSONL or run your own collector.
  • cost.usage is an estimate calibrated to API pricing, not actual billing.
  • Cache-hit rate is the meaningful efficiency metric — cache reads consume less of the rate-limit allocation than fresh input tokens.

Privacy gates (off by default)

  • Prompt text — OTEL_LOG_USER_PROMPTS (else only prompt_length).
  • Tool details — OTEL_LOG_TOOL_DETAILS (params, bash commands, custom agent names).
  • Raw API bodies — OTEL_LOG_RAW_API_BODIES (full conversation history).
  • Extended-thinking content is always redacted, regardless of any flag.
