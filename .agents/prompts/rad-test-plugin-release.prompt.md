# Rad Test Plugin Release — Local Dogfood Marketplace

You are staging the `rad-orchestration` Claude plugin as a **local, ephemeral marketplace** so the user can dogfood it via `/plugin marketplace add` + `/plugin install` before publishing to npm. You do **not** install or verify the plugin yourself — that happens in a separate Claude Code session the user opens manually.

This prompt is the plugin-channel counterpart to `rad-test-release.prompt.md` (legacy `npx radorch` installer). The two are independent — pick the one matching the channel you want to validate.

Run all commands from the repository root. The repo root is the current working directory at the time you start the prompt — capture it (`git rev-parse --show-toplevel` or the cwd) and use that absolute path in the handoff instructions you print to the user.

---

## Step 1 — Build the plugin

From the repo root:

```
npm run build:plugin
```

> Expected: exit 0; `cli/dist/marketplaces/claude/plugins/rad-orchestration/` populated; `validatePluginTree` inside `scripts/build-plugin.js` reports no missing artifacts.

If the build fails, stop and report the failure. Do not continue.

---

## Step 2 — Verify the staged plugin tree and capture the version

Confirm that `cli/dist/marketplaces/claude/plugins/rad-orchestration/.claude-plugin/plugin.json` exists. Read its `version` field and remember it — you will print this version in the handoff message so the user can confirm it after install.

If the file is missing, stop and report. Build must have silently produced an incomplete tree.

---

## Step 3 — Write the ephemeral dogfood marketplace.json

Write this exact JSON to `cli/dist/marketplaces/claude/.claude-plugin/marketplace.json` (create the `.claude-plugin/` directory if it does not exist):

```json
{
  "name": "rad-orchestration-dogfood",
  "owner": { "name": "metalhexx" },
  "description": "Ephemeral local dogfood marketplace for rad-orchestration plugin. Generated by rad-test-plugin-release prompt; do not publish.",
  "plugins": [
    {
      "name": "rad-orchestration",
      "source": "./plugins/rad-orchestration",
      "description": "Rad Orc multi-agent orchestration system for Claude Code."
    }
  ]
}
```

After writing, parse the file back as JSON to confirm it is syntactically valid.

The marketplace name is intentionally distinct from any future published marketplace (`rad-orchestration-marketplace` at the repo root) so transcripts make it obvious which channel was tested.

---

## Step 4 — Print install instructions for the user

Print a block like the following to the user, substituting `<REPO_ROOT>` with the absolute repo-root path you captured at the start (Windows paths with backslashes; the user is on Windows):

> The dogfood marketplace is staged. Plugin version: **`<VERSION>`** (read from `plugin.json` in Step 2).
>
> Open a **fresh Claude Code session** in a test workspace of your choice (any directory — does not need to be related to this repo), then run:
>
> ```
> /plugin marketplace add <REPO_ROOT>\cli\dist\marketplaces\claude
> /plugin install rad-orchestration@rad-orchestration-dogfood
> ```
>
> Reload Claude Code if prompted. The SessionStart hook will auto-bootstrap `~/.radorc/` on next session start.
>
> Come back here when you're done dogfooding — I'll offer cleanup.

---

## Step 5 — Wait for the user, then offer cleanup

After printing the instructions above, your next turn waits for the user to say they're done. When they return, call `AskUserQuestion` to offer cleanup. Surface both options explicitly — do not rely on an auto-injected "Other":

- **Question:** "Done dogfooding the plugin? You can remove the staged plugin tree and the dogfood marketplace, or leave them in place for another round of testing."
- **Options:**
  - **Yes, clean up** — "Remove `<REPO_ROOT>\cli\dist\marketplaces\claude` (deletes both the staged plugin tree and the dogfood marketplace.json). Next `npm run build:plugin` will regenerate."
  - **Leave it for another round** — "Keep the staged tree and marketplace.json so you can iterate (rebuild → reinstall in the test session). Print the cleanup command for later."

If the user picks **clean up**:

```
Remove-Item -Recurse -Force <REPO_ROOT>\cli\dist\marketplaces\claude
```

Run it. Confirm the directory is gone. Print "Cleaned up. The next `npm run build:plugin` will regenerate the staged tree; re-run this prompt to regenerate the dogfood marketplace.json."

If the user picks **leave it**:

Print the cleanup command above so they can run it later. Also remind them that the user is still responsible for running `/plugin uninstall rad-orchestration` (and optionally `/plugin marketplace remove rad-orchestration-dogfood`) inside the test Claude Code session.

That ends the prompt. Do not run tests, npm pack, or any other E2E validation — those belong in CI or a separate workflow.
